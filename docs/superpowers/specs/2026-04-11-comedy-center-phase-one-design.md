# 马达的喜剧中心第一期设计

## 背景

当前项目是 XYSG / TicketSave 的公开票根墙和单管理员管理端。公开端包含 `/` 票根墙和 `/shows/:id` 演出详情；管理端 `/admin` 支持演出、演员、厂牌、场地和 iOS schema v2 ZIP 备份导入导出。

这次产品方向是把站点从“票根展示墙”升级为“马达的喜剧中心”。它会逐步承载首页、票根、演出日历、留言板、喜剧日记、朋友关系和后台管理。第一期先搭好中心站点骨架，并实装首页、票根模块和演出日历。

## 分期规划

第一期：中心站点骨架。将 `/` 改为首页；将当前票根墙迁移到 `/tickets`；保留 `/shows/:id` 票根详情；新增 `/calendar` 演出日历；为 `/guestbook`、`/diary`、`/friends` 提供入口和占位页；后台新增日历管理。

第二期：内容互动。上线留言板、喜剧日记、文章详情、点赞、评论，以及对应后台管理。分页、异步加载、基础审核或风控在这一期处理。

第三期：马达和他的朋友们。上线朋友列表、朋友详情、朋友资料、朋友相册和基于票根/实体关系的同台次数、首次同台、共同厂牌/场地等关系分析。

## 第一期功能范围

第一期包含：

- 首页：网站介绍、更新动态、模块入口、设置与后台管理入口。
- 票根模块：当前票根墙迁移到 `/tickets`，票根详情 `/shows/:id` 保留。
- 演出日历：公开端支持月历视图和列表视图，可切换月份。
- 后台日历管理：支持 JSON 示例下载、JSON 导入、手动新增、编辑、删除。
- 一键生成票根：从日历事件生成标准票根，并回填关联，避免重复生成。
- 预留入口：留言板、喜剧日记、朋友关系模块先提供占位页。

第一期不包含：

- 留言发布、留言分页、留言审核。
- 日记文章管理、文章详情、点赞和评论。
- 朋友详情、相册、同台关系计算。
- 票价、购票链接、日历事件是否公开等售票或发布状态字段。

## 视觉方向

首页和公开端整体气质为明亮、温暖、叙事、有呼吸感、优雅、现代、简约但大气。页面应该像“马达的喜剧生活入口”，而不是工具面板或营销落地页。

视觉系统使用轻量玻璃质感、blur 和柔和轻盈阴影。玻璃质感主要用于导航、精选模块、更新动态、入口卡片和局部浮层；背景保持明亮、温暖、低饱和。文字可读性优先，blur 只作为质感，不作为遮挡内容的主要手段。避免厚重暗色后台感、花哨特效、大面积复杂渐变和单一橙棕或米色铺满。

## 路由结构

- `/`：马达的喜剧中心首页。
- `/tickets`：票根精选和完整票根墙。
- `/shows/:id`：票根详情，沿用现有路径。
- `/calendar`：演出日历公开页。
- `/guestbook`：留言板占位页，第二期实现。
- `/diary`：喜剧日记占位页，第二期实现。
- `/friends`：朋友关系占位页，第三期实现。
- `/admin`：后台管理，新增日历 tab。

## 首页内容

首页包含四个层级：

1. 网站介绍：用叙事化文案说明这里是马达的演出、票根、日记、留言和朋友关系的总入口。
2. 更新动态：第一期从票根和日历数据中展示最近更新，例如最近新增票根、本周演出、近期排期。第二期后可接入最新日记和留言。
3. 模块入口：票根精选、本周演出日历、留言精选、日记精选，以及朋友关系入口。未实现模块显示为即将开放，但入口位置保持稳定。
4. 设置与后台入口：提供 `/admin` 入口，视觉上轻量处理，不抢占主要叙事内容。

## 票根模块

当前公开票根墙从 `/` 迁移到 `/tickets`。票卡、筛选、详情页和公开 API 保持现有能力；首页只展示精选或摘要入口，不承载完整票根墙。

为了兼容旧访问，`/shows/:id` 继续保持不变。是否需要从 `/` 自动跳转到 `/tickets` 不在第一期默认范围，因为 `/` 将承载新的首页。

## 日历数据模型

新增 `calendar_events` 表，日历事件作为排期源，独立于票根 `shows`。日历和票根通过可选 `createdShowID` 关联。

字段：

- `id`
- `title`
- `eventDate`：`YYYY-MM-DD`
- `startTime`：`HH:mm`
- `brandID`
- `venueID`
- `format`
- `myRole`
- `showType`
- `notes`
- `source`：`manual` 或 `import`
- `createdShowID`
- `createdAt`
- `updatedAt`

日历沿用票根的三类枚举，便于一键生成票根：

- `format`：单口、漫才、即兴、新喜剧、其他。
- `myRole`：主持、演员、主咖、开场、其他。
- `showType`：开放麦、商演、主打秀、专场、比赛、其他。

其中“开场”和“比赛”需要同步扩展现有票根枚举，保证日历事件生成票根时不丢语义。

## JSON 导入

后台日历管理页提供“下载 JSON 示例”入口。示例内容：

```json
[
  {
    "date": "2026-04-18",
    "startTime": "20:00",
    "brand": "某某喜剧",
    "venue": "某某剧场",
    "city": "上海",
    "format": "单口",
    "myRole": "主持",
    "showType": "开放麦",
    "title": "周六开放麦",
    "notes": "可选备注"
  }
]
```

主要字段为 `date`、`startTime`、`brand`、`venue`、`format`、`myRole`、`showType`。`city`、`title`、`notes` 可选。导入时按厂牌名称匹配 `brands`，按场地名称和城市匹配 `venues`；找不到时自动创建实体，并在导入结果中列出新增的厂牌和场地。

导入结果包含导入数量、跳过数量、新增厂牌、新增场地和错误行。JSON 格式错误、日期时间不合法、枚举无法识别时返回具体行号和字段说明。

## 日历公开端

`/calendar` 提供月历和列表两种视图。两种视图使用同一份月度 API 数据。

月历视图显示当前月份网格，在有演出的日期做标记。点击日期后显示当天事件详情。

列表视图按日期和开始时间展示当月所有演出。用户可以切换上个月、下个月和回到当月。

首页的“本周演出日历”从日历数据中提取近期演出作为精选入口。

## 后台日历管理

`/admin` 新增“日历”tab。该页面包含：

- 事件列表：默认按日期和开始时间升序展示未来事件，历史事件仍可在列表中查看。
- 手动新增/编辑表单：维护标题、日期、开始时间、厂牌、场地、形式、角色、类型和备注。
- JSON 示例下载。
- JSON 导入控件和导入结果摘要。
- 单条事件删除。
- 单条事件一键生成票根。

手动维护时厂牌和场地使用已有实体选择。若需要补录新厂牌或新场地，维护人员可先在现有实体管理 tab 中创建；JSON 导入仍支持自动创建缺失实体。

## 一键生成票根

后台日历事件提供“生成票根”动作。生成规则：

- `eventDate + startTime` 合并为票根 `date`。
- `title` 写入票根标题；如果日历标题为空，则使用厂牌、场地或类型组合生成可读标题。
- `brandID`、`venueID` 直接写入票根。
- `format`、`myRole`、`showType` 直接写入票根。
- `notes` 写入票根备注，`notesPublic` 默认为 `false`。
- `status` 默认为 `published`，保持现有票根创建行为。
- 封面、演员阵容、标签和成就不由日历生成，后续在票根编辑中补充。
- 生成成功后回填 `createdShowID`，再次点击时提示已生成并提供查看或编辑已有票根的路径。

## API 设计

公开端：

- `GET /api/public/calendar?month=2026-04`：返回该月日历事件。
- `GET /api/public/calendar/upcoming?days=7`：返回首页使用的近期演出摘要。

后台：

- `GET /api/admin/calendar`：获取后台日历事件列表。
- `POST /api/admin/calendar`：手动新增日历事件。
- `PUT /api/admin/calendar/:id`：编辑日历事件。
- `DELETE /api/admin/calendar/:id`：删除日历事件。
- `GET /api/admin/calendar/import-template`：下载 JSON 示例。
- `POST /api/admin/calendar/import`：导入 JSON。
- `POST /api/admin/calendar/:id/create-show`：从日历事件生成票根。

## 测试计划

后端测试：

- 日历事件创建、编辑、删除、按月查询。
- JSON 导入成功路径：匹配已有实体、自动创建缺失厂牌和场地、返回导入汇总。
- JSON 导入失败路径：非法 JSON、缺失必填字段、非法日期时间、无法识别枚举。
- 一键生成票根：字段映射正确、回填 `createdShowID`、重复生成被阻止。
- 公开日历 API 只返回目标月份事件。

前端测试：

- 首页渲染模块入口和近期动态。
- 票根墙在 `/tickets` 可访问，票根详情路径保持可访问。
- 日历公开页能切换月历和列表视图。
- 后台日历管理能展示导入入口、JSON 示例入口和生成票根按钮的状态。

- 打开 `/` 能看到新的“马达的喜剧中心”首页。
- 打开 `/tickets` 能使用原有票根墙能力。
- 打开 `/calendar` 能查看当前月份演出，并在月历和列表之间切换。
- 后台可以下载 JSON 示例、导入 JSON、手动新增/编辑日历事件。
- 缺失厂牌和场地在导入时自动创建，导入完成后能看到汇总。
- 日历事件可以生成票根，生成后的票根可在票根墙和详情页看到。
- 未实现模块有清晰入口和占位状态，不会误导为已上线功能。
